DEPLOYMENT PATHS

三种后端启动方式，按使用场景选

Recommended

方式 A: 本机 PostgreSQL + Docker 后端容器

最适合你自己开发和快速联调。数据库继续放在本机， 后端通过 Docker 保持环境统一。 

docker pull xanxushu/always-backend:latest

docker run --rm -p 3000:3000 ^
  -e DATABASE_URL="postgresql://postgres:postgres@host.docker.internal:5432/always" ^
  -e JWT_ACCESS_SECRET="your-access-secret" ^
  -e JWT_REFRESH_SECRET="your-refresh-secret" ^
  xanxushu/always-backend:latest
  • 最快
  • 最适合单机开发
  • 真机和模拟器都方便测试
Shareable

方式 B: Docker Compose 同时启动 backend + postgres

适合分享给同事、测试同学或者异地机器。整个栈一起起， 最不容易遗漏依赖和网络配置。 

cd always_backend
docker compose up -d

# or force Docker Hub image
$env:ALWAYS_BACKEND_IMAGE="xanxushu/always-backend:latest"
docker compose pull backend
docker compose up -d
  • 最适合协作
  • 数据库卷持久化
  • 部署和迁移路径清晰
Explicit

方式 C: 两个独立容器 + 自定义 Docker 网络

适合你想更细地理解 Docker 网络，或者希望单独控制 PostgreSQL 与 backend 生命周期的时候。 

docker network create always-network

docker run -d --name always-postgres ^
  --network always-network ^
  -e POSTGRES_DB=always ^
  -e POSTGRES_USER=postgres ^
  -e POSTGRES_PASSWORD=postgres ^
  postgres:16-bookworm

docker run -d --name always-backend ^
  --network always-network ^
  -p 3000:3000 ^
  -e DATABASE_URL="postgresql://postgres:postgres@always-postgres:5432/always" ^
  -e JWT_ACCESS_SECRET="your-access-secret" ^
  -e JWT_REFRESH_SECRET="your-refresh-secret" ^
  xanxushu/always-backend:latest
  • 控制最细
  • 适合排查网络
  • 更接近生产拆分思路

HOW TO RUN

从拉镜像到真机联调的实际流程

1

拉取镜像

使用 `docker pull xanxushu/always-backend:latest` 拉取后端镜像。

2

准备数据库

可以选择本机 PostgreSQL、Compose 内 PostgreSQL，或者独立数据库容器。

3

传入环境变量

至少要有 `DATABASE_URL`、`JWT_ACCESS_SECRET`、`JWT_REFRESH_SECRET`。变量多时建议用 `--env-file`。

4

启动容器并看日志

用 `docker ps` 和 `docker logs` 确认后端已经监听到 `3000` 端口。

5

连接 Android

模拟器一般用 `10.0.2.2:3000`，真机用电脑局域网 IP，例如 `192.168.x.x:3000`。
