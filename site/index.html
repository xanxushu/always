INTRODUCTION

01

自建部署

通过Docker在任何地方搭建属于自己的服务， 数据完全掌握在自己手中。

02

关系绑定

03

05

06

DEPLOYMENT PATHS

两种后端启动方式，按使用场景选

Recommended

方式 A: 本机 PostgreSQL + Docker 后端容器

最适合你自己开发和快速联调。数据库继续放在本机， 后端通过 Docker 保持环境统一。 

docker pull xanxushu/always-backend:latest
docker run --rm -p 3000:3000 ^
  -e DATABASE_URL="postgresql://postgres:postgres@host.docker.internal:5432/always" ^
  -e JWT_ACCESS_SECRET="your-access-secret" ^
  -e JWT_REFRESH_SECRET="your-refresh-secret" ^
  xanxushu/always-backend:latest
  • 最快
  • 最适合单机开发
  • 真机和模拟器都方便测试
Shareable

方式 B: Docker Compose 同时启动 backend + postgres

适合分享给同事、测试同学或者异地机器。整个栈一起起， 最不容易遗漏依赖和网络配置。 


# 在项目根目录下创建 docker-compose.yml，内容如下：
services:
  postgres:
    image: postgres:16-bookworm
    restart: unless-stopped
    environment:
      POSTGRES_DB: always
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres
    ports:
      - '5432:5432'
    volumes:
      - postgres-data:/var/lib/postgresql/data

  backend:
    image: xanxushu/always-backend:latest
    restart: unless-stopped
    depends_on:
      - postgres
    env_file:
      - .env.example
    environment:
      DATABASE_URL: postgresql://postgres:postgres@postgres:5432/always
      PORT: 3000
    ports:
      - '3000:3000'

volumes:
  postgres-data:
# 执行compose命令                
docker compose up -d
  • 最适合协作
  • 数据库卷持久化
  • 部署和迁移路径清晰

HOW TO RUN

从拉镜像到真机联调的实际流程

1

拉取镜像

使用 `docker pull xanxushu/always-backend:latest` 拉取后端镜像。

2

准备数据库

可以选择本机 PostgreSQL、Compose 内 PostgreSQL，或者独立数据库容器。

3

传入环境变量

至少要有 `DATABASE_URL`、`JWT_ACCESS_SECRET`、`JWT_REFRESH_SECRET`。变量多时建议用 `--env-file`。

4

启动容器并看日志

用 `docker ps` 和 `docker logs` 确认后端已经监听到 `3000` 端口。

5

连接 Android

真机用电脑局域网 IP，例如：192.168.x.x:3000。
